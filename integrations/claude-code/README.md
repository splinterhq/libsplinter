# Claude Code & Splinter Integration Demo

This was originally a companion repository for a dev.to post on how 
to teach Claude agents to use Splinter's command-line implementation
to signal their intent in advance, as well as journal their activity
as far as expectations vs reality / trial and error so that there's
an immediate semantically-accessible memory of what happened during 
the run for both the agents and the user running them. 

It's not quite nailed down enough yet to be a *lasting* artifact that
doesnt itself require constant upkeep, so I'm including it in this 
repo just so it can collect some eyeballs and maybe see some improvement.

There's an LCARS-style display to visualize Splinter's signal groups in 
the `ts/` directory (Deno required). 

Take a look. PRs welcome. Just remember it's incomplete and the agents
drift from instructions a bit.
